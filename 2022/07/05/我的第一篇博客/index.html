利用hexo创建博客并部署远端（Windows版本）


阅读数: 次    2022-07-05

现有分享大多是基于Linux下进行博客的创建，在codesheep的讲解下，自己总结分享一下Windows下利用hexo创建博客并部署到远端（免费：部署到GitHub）

第一章.介绍

  1. Hexo是一个快速、简介且高效的博客框架。使用Markdown解析文章，且速度很快，几秒内就可利用靓丽的主题生成静态网页。更详细的介绍可见https://www.jianshu.com/p/e4f06f5a1f62/
  2. Node.js是创建博客需要搭配的软件，这是运行在服务端的一个JavaScript。下载地址为https://nodejs.org/en/,下载安装成功后有两个组件，分别是Nodejs和npm。
  3. Vim是一个类似于Vi的著名的功能强大、高度可定制的文本编辑器，通俗点类似于Word、记事本等文本编辑器。
  4. Git是分布式版本控制系统，git通常在编程中会用到，并且git支持分布式部署，可以有效、高速的处理从很小到非常大的项目版本管理。分布式相比于集中式的最大区别在于开发者可以提交到本地，每个开发者通过克隆（git clone），在本地机器上拷贝一个完整的Git仓库。
    参考文章https://www.php.cn/tool/git/412871.html 举例子：比如当你在本地写好某一个编程文档时，发现有些地方需要修改或者删除，有的人可能会直接在当前文件中直接修改，有的人会复制一份在上面修改，然后删除没用的文件。但是当你发现还是原来的文件好或者另外的版本好时，就可能手足无措了。此时使用git工具，就是聪明之举了。我们可以在本地建一个版本库（仓库即repository，对应GitHub上面的仓库），每当我们需要修改时，就可以把之前的版本提交并标明此版的特点。这样文件夹里就只有一个编程文档了。当你需要哪个版本时，只要在版本库中恢复一下就可以了。

第二章.安装软件、创建博客并部署远程

  1. Vim在安装时要注意安装路径，后期需要自己配置环境变量。例如我的Vim安装路径为D:\Blog\Vim，然后安装好后，找到D:\Blog\Vim\vim81这个路径，复制粘贴来配置环境路径。
    -有想插入本地图片，如上所示，需要：1.修改博客根目录下的_config.yml的配置： post_asset_folder:true ; 2.然后找到存放博客的文件夹，我这里是D:\Blog\blog，找到source\_posts文件夹，这里就是博客。创建一个文件夹，与博客同等级且文件名为博客文件名，里面放置要引入的图片；3.在文章正常引用：
  2. 打开终端Win+R，输入cmd，切换到root用户权限。命令是runas /user:用户名 cmd
    或者直接搜索框搜索cmd，然后以管理员身份运行即可。
  3. 可以用命令node -v查看安装好的Nodejs的组件;用命令npm -v查看安装好的npm组件。这里如果有错误，提示 --global的错误，请找到D:\Blog\nodejs文件下的nmp.cmd文件，将提示错误地方改为prefix --location=global即可
  4. 接下来借助淘宝镜像源来安装hexo博客框架。首先输入命令npm install -g cnpm --registry=https://registry.npm.taobao.org，等待cnpm下载，下载好后输入命令cnpm -v查看是否下载安装成功，成功则显示对应版本。接着输入命令cnpm install -g hexo-cli来安装hexo框架，可以通过hexo -v来查看是否安装成功。新建一个空文件夹来存放后期生成的博客，为方便，我在D:\Blog下使用命令mkdir blog来新建文件夹blog,然后使用命令cd blog/进入blog文件夹。
  5. 进入blog文件夹，使用命令hexo init生成博客，然后使用命令hexo -s来启动博客，接着使用命令hexo n “博客名”来新建自己的第一个博客。
  6. 使用命令vim 博客名.md来编辑新建的第一个博客。按i切换为插入模式开始编辑，编辑结束按esc，然后冒号:，接着输入wq保存编辑的内容并退出编辑；!q是不保存并退出编辑。具体编辑方法可参考https://www.runoob.com/markdown/md-tutorial.html
  7. hexo clean
  8. hexo g
  9. hexo s 此时就能在本地查看自己的博客了，一般是http://localhost:4000，复制粘贴到浏览器即可

激动人心的时刻要到了！

  1. 输入git config --global user.name "cauc01" git config --global email.name "xiaodongy9634@163.com"

然后在D:\Blog\blog\source下右键鼠标，点击Git Bash here
依次输入命令

  • hexo g
  • hexo s
  • hexo d

最后返回终端，在blog下依次键入

  • hexo g
  • hexo s
  • git config --global user.name "cauc01"
